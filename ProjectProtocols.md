## Анализ модулей и зависимостей

Проект построен по модульному принципу, где каждый компонент выполняет строго определенную роль. Основной модуль `Program.cs` выступает в роли координатора.

### Таблица зависимостей модулей

| Модуль | Вызываемый метод | Назначение вызова | Передаваемые данные |
| :--- | :--- | :--- | :--- |
| **Program.cs** | `InputModule.ReadString()` | Получение строки от пользователя | Текст приглашения (string) |
| **Program.cs** | `ValidationModule.IsNotEmpty()` | Проверка на пустую строку | Введенная строка (string) |
| **Program.cs** | `ProcessingModule.CountAndPrintVowels()` | Обработка и вывод результата | Проверенная строка (string) |

### Описание взаимодействия
Процесс работы программы начинается в модуле `Program`, который запрашивает ввод у пользователя через `InputModule`. Полученная строка передается в `ValidationModule` для проверки на корректность (отсутствие пустого ввода). В случае успешной валидации строка направляется в `ProcessingModule`, который выполняет алгоритм подсчета гласных букв и выводит результат на экран. Такая структура обеспечивает слабую связанность компонентов и высокую читаемость кода.

## Примечания и рекомендации
- **Улучшение:** В `ValidationModule` можно добавить дополнительные проверки, например, на наличие цифр или специальных символов, если это потребуется по условию задачи.
- **Расширение:** `ProcessingModule` может быть дополнен методами для подсчета согласных букв или знаков препинания без изменения логики других модулей.
